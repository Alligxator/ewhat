# 🍜 Ewhat — 今天吃什么 产品需求文档 (PRD)

> 为选择困难症而生的美食随机推荐 App

---

## 一、产品定位

**一句话描述：** 用抽卡 + 玄学的方式，帮用户解决"今天吃什么"的终极难题。

**目标用户：** 每天纠结吃什么的都市打工人、学生党、情侣/室友决策困难户

**核心调性：** 趣味 > 工具，仪式感 > 效率，精致 > 花哨

**平台：** iOS (SwiftUI, 最低支持 iOS 17)

---

## 二、核心功能

### 2.1 🎴 抽卡推荐（主玩法）

**交互流程：**
1. 用户进入首页，看到一张待翻转的美食卡牌（背面朝上，带呼吸光效）
2. 长按/上滑触发抽卡动画（卡牌翻转 + 粒子特效）
3. 翻转后展示推荐结果：美食名称 + 精美插图 + 一句趣味文案
4. 底部两个按钮：
   - **「就吃这个！」** → 确认选择，触发撒花/烟花庆祝动画，记录到今日饮食
   - **「朕不喜欢 👎」** → 卡牌飞出屏幕，自动抽下一张（有抽卡次数提示，如"第3次翻牌"）

**卡牌设计：**
- 每张卡牌有独特的视觉风格（根据菜系/食物类型区分配色）
- 卡牌等级：普通 / 稀有 / 传说（纯趣味，如"传说级·佛跳墙"）
- 翻牌动画需流畅、有质感（参考原神/阴阳师抽卡手感）

### 2.2 🔮 日历时运系统

**玄学加成（趣味性，非迷信）：**
- 根据当日农历/节气/星期，生成「今日食运」：
  - 例："宜辣忌甜"、"今日火属性旺，推荐烧烤/火锅"
  - 节气特色："立秋宜贴秋膘 → 大肉推荐权重 UP"
  - 星期惯例："周五解放日 → 推荐聚餐/大餐"
- 首页顶部展示今日食运卡片（农历日期 + 食运签文 + 推荐属性）
- 用户可选择是否让食运影响推荐结果（开关控制）

### 2.3 🎯 推荐模式选择

进入抽卡前，用户可选择筛选维度：

**模式一：按菜系**
- 中餐（川菜/粤菜/湘菜/东北菜/江浙菜/西北菜...）
- 日料 / 韩餐 / 东南亚 / 西餐 / 快餐
- 「全部随机」（不限菜系）

**模式二：按食物类别**
- 麻辣烫 / 火锅 / 面条 / 米饭盖浇 / 饺子包子 / 烧烤 / 炒菜
- 汉堡 / 披萨 / 寿司 / 沙拉 / 粥 / 小龙虾...

**模式三：按场景**
- 一个人随便吃 / 朋友聚餐 / 约会 / 夜宵 / 下午茶

**模式四：按预算**
- 穷鬼模式 (< ¥20) / 正常 (¥20-50) / 小奢侈 (¥50-100) / 随便花

> 模式可叠加组合，如"川菜 + 一个人 + 穷鬼模式"

### 2.4 📊 饮食记录（轻量）

- 每次「就吃这个」自动记录到日历
- 周/月维度统计：吃过的菜系分布饼图、最爱 Top 5、连续吃同类的"上瘾警告"
- 「最近吃过」自动降低推荐权重（避免连续推荐同一个）

## 三、补充功能

### 3.1 🎰 多人决策模式（帮朋友/情侣做决定）

- 「帮我们选」模式：多人各自设置偏好/排除项 → 系统取交集随机
- 转盘模式：生成一个美食转盘，所有人一起看结果（适合投屏/截图分享）

### 3.2 🚫 黑名单 & 偏好设置

- 永久排除：不吃的食物/菜系（如过敏、宗教、素食）
- 临时排除："今天不想吃辣" → 本次会话内屏蔽辣味相关
- 偏好加权：喜欢的类型提高出现概率

### 3.3 🏆 趣味成就系统

- 「美食探险家」— 解锁过 50 种不同食物
- 「忠实粉丝」— 同一食物选了 10 次
- 「选择恐惧终结者」— 首次抽卡就确认
- 「朕很难伺候」— 连续拒绝 10 次
- 成就解锁时弹出精美徽章 + 动画

### 3.4 📤 分享功能

- 一键生成精美分享卡片（今日推荐 + 食运签文）
- 支持分享到微信/朋友圈/社交平台

### 3.5 🔔 每日推送（可选）

- 午餐前/晚餐前推送："今日食运已更新，来抽一卡？"
- 推送文案随机趣味化

## 四、UI/UX 设计要求

### 4.1 整体风格

- **设计语言：** 新中式 + 卡通质感，精致但不沉重
- **配色：** 暖色调为主（食欲色系），深色/浅色模式自适应
- **字体：** 中文用圆体/手写体（标题），正文用系统默认保证可读性
- **插图：** 每种食物配手绘风格插图（可先用 SF Symbols + emoji 占位，后期替换）

### 4.2 动画要求

- **抽卡翻转：** 3D 翻转 + 光晕，时长 ~0.6s，有弹性回弹
- **拒绝动画：** 卡牌向左飞出 + 缩小消失，新卡从右侧滑入
- **确认动画：** 彩色粒子爆炸 / 满屏撒花
- **食运卡片：** 缓慢浮动呼吸效果
- **页面转场：** 流畅的 matched geometry 过渡

### 4.3 关键页面

1. **首页：** 今日食运 + 抽卡入口（最核心，要一眼抓住）
2. **模式选择页：** 筛选条件的优雅展示（标签流/网格布局）
3. **抽卡结果页：** 大卡牌 + 按钮（沉浸感）
4. **饮食记录页：** 日历视图 + 统计图表
5. **设置页：** 偏好/黑名单/成就

### 4.4 交互细节

- 触觉反馈（Haptics）：抽卡、确认、拒绝都有对应震动
- 音效（可选开关）：翻牌音效、确认欢呼
- 手势支持：左滑拒绝 / 右滑确认 / 长按查看详情

## 五、技术方案

### 5.1 技术栈

- **语言：** Swift
- **UI 框架：** SwiftUI (iOS 17+)
- **架构：** MVVM + Swift Observation (@Observable)
- **数据持久化：** SwiftData（饮食记录、偏好设置）
- **动画：** SwiftUI Animation + Canvas API（粒子效果）
- **日历计算：** Foundation Calendar（农历/节气）

### 5.2 数据结构

- **食物数据库：** 本地 JSON，包含 200+ 食物条目
  - 字段：名称 / 菜系 / 类别 / 标签 / 场景 / 价格区间 / 图片 / 趣味文案
- **食运算法：** 基于农历日期 + 节气 + 星期的确定性伪随机（同一天同一配置 = 同一食运）
- **推荐算法：** 加权随机（偏好加权 + 近期去重 + 食运加成）

### 5.3 无需后端

- V1 纯本地运行，无需联网
- 食物数据库内置在 App 中
- 所有数据存储在设备本地
